# Progress Reporting

> **Evidence Status** — synthesized. 长任务、Coding Agent、Research Agent、Workflow Agent 的进度交付需求。

## 1. 报告触发

| 触发 | 报告内容 |
|---|---|
| milestone 完成 | 完成了什么、证据、下一步 |
| 长时间无用户可见输出 | 当前阶段、阻塞点、是否需要用户 |
| 失败恢复 | 失败类型、已尝试、改用策略 |
| 成本接近预算 | 已消耗、剩余选项 |
| 需要等待外部系统 | 等待对象、可继续做什么 |

## 2. 报告模板

```text
当前进度：...
已验证：...
正在处理：...
需要你决定：...（如果没有就省略）
风险/不确定性：...（如果没有就省略）
```

## 3. 粒度控制

- 低风险短任务：不主动报告，只最终交付。
- 长任务：milestone 报告。
- 高风险任务：关键 gate 报告。
- 流式/持续任务：按事件严重度报告，而不是按时间刷屏。
