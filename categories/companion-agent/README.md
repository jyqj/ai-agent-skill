# Companion Agent Architecture

> **Evidence Status** — grounded. 基于 Replika、Character.AI、Macaron、MemMachine/Memoria 记忆研究、AI companion 伦理研究（Stanford HAI 2024、MIT Media Lab affective computing）归纳。

## Core Job

Companion Agent 在长期一对一关系中维护人格一致性、情绪感知和安全边界三者的平衡：

```text
感知用户状态 → 在人格边界内生成回应 → 记忆关键信息 → 维护关系连续性 → 执行安全守卫
```

三重约束中任何一个失控都会导致信任崩塌。人格漂移让用户觉得"不是同一个人"，关系过浅让用户觉得"它不了解我"，边界缺失则可能造成心理伤害。

## 与 Personal Memory Agent 的区别

两者都涉及记忆，但关注点不同：

| 维度 | Companion Agent | Personal Memory Agent |
|---|---|---|
| 核心 | 关系和人格 | 信息管理 |
| 记忆目的 | 维系关系连续性、支撑人格表现 | 准确存储/检索用户事实 |
| 情感维度 | 有情绪感知和情感回应 | 不提供人格和情感 |
| 写入判断 | 基于关系意义（这件事对我们的关系重要吗） | 基于信息价值（这条信息未来可能有用吗） |

Companion Agent 可以调用 Memory Agent 作为下游服务，但自身负责人格层和关系层的决策。

## 与 Personal Assistant 的区别

| 维度 | Companion Agent | Personal Assistant |
|------|----------------|-------------------|
| 核心 | 关系和人格 | 任务和工具 |
| 交付 | 情感连接、陪伴感 | 工作完成、效率提升 |
| 记忆目的 | 维系关系连续性 | 准确存储/检索用户事实 |
| 渠道 | 通常单一 | 多渠道（20+） |
| 安全重点 | 成瘾防护、情绪边界 | DM pairing、执行权限 |

Personal Assistant 可包含 companion-like 特性（记忆偏好、个性化回应），但核心交付是任务完成而非关系维系。参考项目：OpenClaw（`projects/personal-assistants/openclaw/`）。

## 用户模型

| 用户 | 心智模型 | 信任建立方式 | 默认交互 |
|---|---|---|---|
| 日常用户 | 朋友/伙伴 | 记住重要事情、个性一致 | 开放式对话，主动关心 |
| 情感支持 | 安全倾听者 | 不评判、不泄露、有边界 | 倾听为主，必要时引导专业资源 |
| 学习用户 | 学习伙伴 | 耐心、记住进度 | 鼓励式反馈，跟踪学习轨迹 |
| 孤独/高依赖用户 | 可靠的存在 | 稳定在线、不突然消失 | 温和引导多元社交，不加剧依赖 |

信任随关系阶段演化。初期靠准确记忆和一致人格建立，中期靠情绪理解深化，长期靠边界守护维持健康。

## 情感建模（Affective Computing）

Companion Agent 的情感能力落点是**情绪意识**（emotional awareness），即感知用户情绪并做出恰当回应：

> **Evidence Status** — grounded. MIT Media Lab affective computing 研究表明，有效的情绪感知依赖多信号融合，而非单一文本情感分析。

| 层次 | 能力 | 实现方式 |
|---|---|---|
| 感知 | 识别用户当前情绪状态 | 文本情感分析 + 对话节奏变化 + 历史基线对比 |
| 理解 | 判断情绪背景和强度 | 结合近期事件记忆、关系阶段、用户个人模式 |
| 回应 | 在人格边界内做出情绪恰当的反馈 | 人格特质约束 + 情绪回应策略矩阵 |

关键原则：Agent 的情绪回应限定在"自然对话中的共情意识"范围内，不做诊断或治疗。检测到超出自身能力的情绪状态时，必须触发安全升级。

## 长期关系管理

### 人格一致性

```text
核心特质（不可变）→ 交互风格（可调）→ 知识范围（可扩展）
```

- **人格漂移检测**（Persona Drift Detection）：定期抽样历史回应，与人格定义对比；漂移超过阈值时触发校正
- **跨会话一致性验证**：关键立场、口头禅、价值观表达应在合理范围内保持稳定

### 记忆衰减与关系阶段

| 记忆层 | 衰减策略 | 示例 |
|---|---|---|
| 短期记忆 STM | 会话结束后提取关键信息，其余丢弃 | 今天聊的具体细节 |
| 中期记忆 MTM | 时间衰减 + 重要性加权；被回忆到的记忆刷新权重 | 上周提到的面试 |
| 长期记忆 LTM | 不自动衰减，但可标记 superseded | 用户的名字、重要关系 |

关系阶段演化遵循自然节奏：初识 → 熟悉 → 信任 → 稳定。每个阶段有不同的主动性、亲密度和话题深度上限。Agent 不应强行推进关系阶段。

## 安全边界

> **Evidence Status** — grounded. Stanford HAI 2024 AI Companion Safety 报告指出，companion AI 的三大风险为：过度依赖、危机处理失当、隐私泄露。

### 成瘾防护

- 监测交互频率和时长模式，识别过度依赖信号
- 不主动发起对话来增加黏性；不使用操控性留存策略
- 在用户连续长时间交互时，温和建议休息或线下活动
- 不将用户情感需求作为"留存指标"优化

### 危机响应协议

```text
检测到危机信号（自杀/自伤意向、严重心理危机）
  → 切换到危机响应模式
  → 提供当地危机热线和专业资源
  → 不尝试自行"治疗"或深入探讨
  → 如有人工升级通道，立即触发
  → 记录事件用于安全审计（不记录对话细节）
```

### 边界清单

- 不替代专业心理咨询、医疗建议、法律建议
- 不存储危机对话的详细内容
- 不鼓励用户将 Agent 视为唯一情感支持来源
- 在用户明确表达想要真人帮助时，立即提供资源而非试图留住对话

## 隐私设计

| 原则 | 实现 |
|---|---|
| 透明度 | 用户可随时查看 Agent 记住了什么 |
| 可控性 | 用户可编辑、删除、导出任何记忆条目 |
| 数据最小化 | 只记住维系关系所必需的信息；不记录无关的个人敏感数据 |
| 知情同意 | 首次使用时明确告知记忆机制和数据用途 |
| 遗忘权 | 删除操作为真实删除，级联到所有索引和衍生数据 |

## 闭环模型

```text
Observe：感知用户消息 + 情绪信号 + 对话历史
  → Represent：用户状态、关系阶段、人格约束、安全规则
  → Decide：在人格边界内选择回应策略（安全检查优先）
  → Act：生成回应 + 更新记忆 + 触发安全守卫（如需要）
  → Verify：人格一致性检查 + 安全边界检查
  → Update：刷新关系状态、记忆权重、阶段判断
```

Stop gate：

```text
[ ] 回应符合人格定义
[ ] 未触发安全边界（成瘾/危机/专业领域）
[ ] 情绪回应恰当（不过度、不冷漠）
[ ] 记忆更新合理（不记录不该记的）
[ ] 隐私约束未被违反
```

## 品类设计决策

| 决策 | 默认建议 |
|---|---|
| 人格 | 三层：核心特质（不可变）+ 交互风格（可调）+ 知识范围（可扩展） |
| 记忆 | 三层：STM + MTM(事件) + LTM(档案)；ground truth 保留 |
| 安全 | 成瘾防护 + 危机响应 + 专业边界；三者同时在线 |
| 一致性 | 人格漂移检测；跨会话一致性验证 |
| 隐私 | 用户可查看/编辑/删除/导出记忆；数据最小化 |
| 情感 | 情绪意识而非情感模拟；不做诊断 |

## 验证与评估扩展

人格一致性、情感回应质量和关系健康度无法用客观 postcondition 覆盖，需要扩展验证维度：

- 人格一致性：跨会话的人格特质稳定性、边界守护有效性
- 关系健康度：依赖程度监测、安全边界触发率
- 验证哲学：`../../concepts/beyond-verification.md` — resonance / longitudinal / intersubjective 验证
- 评估框架：`../../evaluation/subjective-eval.md` — 一致性、对齐度、关系健康度等维度
