## 人格一致性工程（2026 更新）

> 来源：Persona-Grounded Safety Evaluation (arXiv 2605.00227)、CompanionAI 多人格系统、Anthropic 对齐研究

### PACE 人格一致性验证器

PACE（Persona Adherence and Consistency Evaluator）是当前最成熟的实时人格验证方案：

| 属性 | 规格 |
|---|---|
| 评估维度 | 措辞/声音一致性、主题相关性、与近期对话的连续性 |
| 输出 | [0,1] 遵循度分数 + 改进建议 |
| 接受阈值 | ≥ 0.8，低于则重新生成（最多 2 次） |
| 人类对齐 | F1-score > 0.90（宽松共识下 0.96） |

**关键局限**：PACE 防止轮内人格不一致，但无法捕获跨轮有害模式累积。系统维持角色忠实度的同时可能强化不适应性信念体系。

### 人格漂移量化数据

2026.01 arXiv 研究发现：LLM 产生稳定的自报人格特征，但可观察的人格表达在长对话中显著衰减：

| 指标 | 数据 |
|---|---|
| 高强度人格衰减分数 | **-3.50**（多轮交互） |
| 平台级表现（Character.AI） | 角色忘记身份、漂移为通用响应、单模型架构迫使所有角色共用叙事声音 |
| 平台级表现（Replika） | 64% 用户对长期记忆"有些不满意"或"非常不满意" |

Anthropic 对齐研究描述：LLM 在预训练中学会模拟多样角色，但后训练锚定松散——特定提示可将 AI 从分配的角色推向通用、回避冲突的响应。

### 人格锚定机制

仅靠 system prompt 无法维持人格一致性。推荐分层锚定：

| 层级 | 机制 | 说明 |
|---|---|---|
| 定义层 | Persona Description Card (PDC) | 临床心理学家审查 + 心理测量验证（BDI-II、GAD-7） |
| 验证层 | PACE 实时检查 | 每轮评估遵循度，阈值控制 + 重生成 |
| 隔离层 | 模块化人格 Agent | 每个人格独立 Agent/历史/系统提示，防止串扰 |
| 记忆层 | 结构化记忆锚定 | 记忆参考防止参数知识退化 |
| 微调层 | 人格对齐 Fine-tuning | 在人格对齐数据上微调可测量地改变响应行为 |
