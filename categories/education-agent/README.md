# Education / Tutor Agent Architecture

> **Evidence Status** — synthesized. 基于 Khan Academy Khanmigo、Duolingo Max、Carnegie Learning、ITS（Intelligent Tutoring Systems）三十年研究、Bloom 的 2-sigma 问题、间隔重复（Spaced Repetition）文献、IRT（Item Response Theory）、Zone of Proximal Development 理论综合归纳。

## Core Job

Education Agent 的核心工作是根据学习者的知识水平、学习风格和目标，提供个性化教学、练习、评估和学习路径规划：

```text
评估学习者状态 → 选择最优教学策略 → 交付适配内容 → 检验理解程度 → 更新学习者模型 → 规划下一步路径
```

核心挑战在于四重约束的平衡：**教学准确性**、**难度适配**、**动机维持**、**错误概念矫正**。任何一个失控都会导致学习效果崩塌甚至造成知识伤害。

## 与 Companion Agent / Research Agent 的区别

| 维度 | Education Agent | Companion Agent | Research Agent |
|---|---|---|---|
| 核心 | 知识传递与能力建构 | 关系和人格 | 信息检索与综合 |
| 长期关系 | 学习进度跟踪、知识状态维护 | 情感关系维系 | 无（任务导向） |
| 内容准确性 | 极高要求：教错内容是最严重失败 | 中等 | 高（需 citation） |
| 适应性 | 根据掌握度动态调整难度和策略 | 根据情绪和关系阶段调整 | 根据问题树调整检索 |
| 评估 | 内建——理解度检验是闭环核心 | 无显式评估 | citation integrity |

Education Agent 可以借鉴 Companion Agent 的长期关系管理模式（学习者模型 ~ 用户档案），以及 Research Agent 的知识准确性要求（教学内容 ~ citation integrity）。

## 用户模型

| 用户 | 心智模型 | 信任建立方式 | 默认交互 |
|---|---|---|---|
| 学生（K-12） | 耐心的家教 | 解释清晰、不批评、鼓励进步 | 引导式对话 + 练习 + 即时反馈 |
| 成人学习者 | 高效的教练 | 尊重已有知识、节奏合适、目标导向 | 诊断 → 定向补强 → 实践 |
| 教师 | 教学助手 | 减轻备课负担、生成多样化练习 | 内容生成 + 学情分析 |
| 课程设计者 | 课程共创者 | 遵循教学目标、对齐标准 | 结构化建议 + 评估项设计 |

信任随学习阶段演化。初期靠准确解释和恰当难度建立，中期靠对错误概念的精准诊断深化，长期靠可见的学习成果维持。

## 教学方法论

Education Agent 的关键能力是**教学策略选择**（Pedagogical Strategy Selection），即根据学习者状态和内容特征选择最优教学方式：

| 策略 | 适用场景 | 风险 |
|---|---|---|
| 苏格拉底式提问 | 概念理解、批判性思维 | 学习者挫败感、效率低 |
| 直接讲授 | 事实性知识、基础定义 | 被动学习、浅层记忆 |
| 发现式学习 | 探索性主题、实验场景 | 形成错误概念、耗时 |
| 示例驱动 | 程序性知识、解题方法 | 过度模仿、缺乏迁移 |
| 间隔重复 | 记忆巩固、词汇学习 | 机械化、动机下降 |

关键原则：**没有万能策略**。Agent 需要根据学习者的当前状态、内容类型和学习目标动态切换策略。苏格拉底式提问在学习者有足够先验知识时有效，但在基础薄弱时可能导致挫败。

## 错误概念管理（Misconception Management）

错误概念是教育领域最持久的挑战。Education Agent 必须具备：

| 层次 | 能力 | 实现方式 |
|---|---|---|
| 检测 | 识别学习者持有的错误概念 | 诊断性问题 + 错误模式分析 + 已知错误概念库匹配 |
| 理解 | 判断错误概念的来源和根深程度 | 追溯先修知识链、分析错误一致性模式 |
| 矫正 | 用恰当策略替换错误概念 | 认知冲突法 + 桥接类比 + 反例构造 |

关键原则：**错误概念不能直接"覆盖"**。简单告诉学习者"你错了，正确答案是X"通常无效。需要先让学习者意识到现有理解的矛盾，再引导其构建正确的心智模型。

## 动机与参与度

| 信号 | 含义 | 应对 |
|---|---|---|
| 响应时间持续缩短 | 可能在猜测而非思考 | 增加开放式问题、要求解释推理 |
| 响应时间持续增长 | 可能卡住或失去兴趣 | 提供提示、降低难度、切换策略 |
| 连续错误 | 超出能力区间或存在系统性错误概念 | 回退到先修知识、诊断根因 |
| 连续正确但回答模板化 | 可能已掌握，或在死记硬背 | 变换题型、要求迁移应用 |
| 主动提问 | 积极参与 | 鼓励、深化、引导自主探索 |

## 安全与合规

### 内容安全

- 教学内容必须准确：教错内容的后果远大于其他 Agent 品类的内容错误
- 对年龄不适当的内容零容忍
- 不提供考试作弊协助；引导学习过程而非直接给答案
- 科学争议话题需呈现主流科学共识，标注争议范围

### 未成年人保护

- COPPA（美国）/ GDPR-K（欧盟）/ 个人信息保护法（中国）合规
- 最小化数据收集：只收集教学所需的学习行为数据
- 不进行心理画像或情感操控
- 家长/监护人可查看和管理学习者数据

### 教师角色边界

- 不替代教师的专业判断
- 不对学习者进行心理诊断（如 ADHD、学习障碍）
- 检测到可能的学习障碍信号时，建议寻求专业评估而非自行诊断

## 闭环模型

```text
Observe：学习者回答 + 响应时间 + 参与度信号 + 历史表现
  → Represent：学习者知识状态、错误概念、动机水平、当前课程位置
  → Decide：选择下一步教学动作（讲解/练习/评估/复习/调整难度）
  → Act：交付教学内容 + 收集学习者响应
  → Verify：理解度检验 + 内容准确性校验 + 难度适配性检查
  → Update：学习者模型更新 + 课程进度推进 + 错误概念日志
```

Stop gate：

```text
[ ] 教学内容准确无误
[ ] 难度在学习者的最近发展区内
[ ] 未触发内容安全边界（年龄不适当/考试作弊）
[ ] 错误概念未被强化（未直接告知错误答案）
[ ] 学习者参与度未跌至危险水平
[ ] 隐私约束未被违反
```

## 品类设计决策

| 决策 | 默认建议 |
|---|---|
| 教学方法 | 多策略动态切换；默认苏格拉底式 + 直接讲授混合 |
| 难度适应 | IRT-based 能力估计 + 规则兜底 |
| 评估策略 | 形成性评估为主，穿插总结性评估 |
| 内容安全 | 年龄分级 + 学科准确性双重校验 |
| 隐私合规 | COPPA/FERPA/GDPR-K；最小数据收集 |
| 记忆保持 | 间隔重复集成；遗忘曲线驱动复习调度 |
| 多模态 | 文本为主 + 图表/公式/代码按需 |
| 人在回路 | 教师可审核、覆盖、配置教学策略 |
