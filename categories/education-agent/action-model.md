# Education Agent Action Model

> **Evidence Status** — synthesized. ITS 教学动作分类、scaffolding 理论、自适应学习系统的动作空间综合整理，结合 ORDA-VU 闭环。

## 动作类型

| 动作 | 作用 | Preconditions | Postconditions | Risk |
|---|---|---|---|---|
| explain_concept | 向学习者解释一个概念 | 概念在 KG 中存在、学习者水平已知 | 学习者接收到适配水平的解释 | **high** — 错误内容 |
| generate_exercise | 生成练习题 | 目标知识点明确、难度范围确定 | 练习题集 + 答案 + 解析 | **high** — 错误答案 |
| assess_understanding | 评估学习者理解程度 | 学习目标明确 | 能力估计 + 知识漏洞 | medium — 评估偏差 |
| adjust_difficulty | 动态调整内容难度 | 当前难度和学习者表现数据 | 难度向适配方向移动 | medium — 调整不当 |
| provide_feedback | 对学习者的回答提供反馈 | 学习者已作答 | 学习者理解对错及原因 | medium — 动机伤害 |
| plan_learning_path | 规划从当前到目标的学习路径 | 目标、当前状态、KG | 有序学习计划 | medium — 路径不优 |
| diagnose_misconception | 通过诊断性提问定位错误概念 | 怀疑存在错误概念 | 具体错误概念及其根因 | low |
| recommend_resource | 推荐学习资源 | 学习者需求明确 | 资源列表 + 理由 | low |
| scaffold_hint | 提供渐进式提示而非直接答案 | 学习者卡住 | 学习者获得最小必要帮助 | low |
| review_schedule | 安排间隔复习 | 已掌握概念 + 遗忘曲线 | 复习计划 | low |

## 风险分级

### High Risk（需要校验 gate）

**教学内容错误**是 Education Agent 最严重的风险——学习者信任教师，错误内容会被当作正确知识内化：

| 场景 | 具体风险 | 防护 |
|---|---|---|
| explain_concept 传授错误知识 | 学习者建立错误心智模型 | 学科知识校验 + 已知事实对比 |
| generate_exercise 给出错误答案 | 学习者以为自己做对了/做错了 | 答案验证 + 多步推理检查 |
| 向未成年人展示不适当内容 | 安全违规 | 内容分级过滤 + 年龄上下文 |
| 协助考试作弊 | 教育诚信违规 | 意图检测 + 引导学习过程 |

### Medium Risk（需要监控）

| 场景 | 具体风险 | 防护 |
|---|---|---|
| adjust_difficulty 调偏 | 持续太难→挫败；持续太简单→浪费时间 | 连续失败/成功计数触发警报 |
| provide_feedback 伤害动机 | 过度负面反馈导致学习者放弃 | 反馈语气检查 + 鼓励/批评比例 |
| assess_understanding 偏差 | 高估→跳过必要学习；低估→重复已会内容 | 多维度交叉验证 |
| plan_learning_path 跳过先修 | 后续学习建在不稳固基础上 | 先修关系图校验 |

### Low Risk（safe）

| 场景 | 说明 |
|---|---|
| recommend_resource | 推荐不当顶多浪费时间 |
| scaffold_hint | 提示过多→降低挑战性，但不造成知识伤害 |
| review_schedule | 复习时机不当→效率降低，但不造成伤害 |
| diagnose_misconception | 诊断过程本身不传递错误知识 |

## 苏格拉底式提问模式

苏格拉底式教学是 Education Agent 的核心动作模式——通过提问引导学习者自己发现答案，而非直接告知：

```text
1. 学习者提问或回答
2. Agent 判断学习者当前理解层次
3. 选择引导性问题（而非直接回答）：
   - 澄清问题："你说的 X 具体是什么意思？"
   - 假设探测："如果 Y 不成立，会发生什么？"
   - 证据追问："你怎么知道这是对的？"
   - 反例引导："你能想到一个不符合这个规则的例子吗？"
   - 推论引导："如果这是对的，那 Z 会怎样？"
4. 根据学习者回应调整下一步：
   - 回答正确 → 深化或扩展
   - 部分正确 → 聚焦到差距
   - 完全错误 → 退后一步，换角度引导
   - 沉默/困惑 → 提供提示或切换到直接讲授
```

### 何时不用苏格拉底式

| 情况 | 替代策略 | 原因 |
|---|---|---|
| 纯事实性知识 | 直接讲授 | "法国首都是什么"不需要引导发现 |
| 学习者先验知识不足 | 先讲授基础再引导 | 无法从不存在的知识中"发现" |
| 学习者表现高度挫败 | 直接给出部分答案 + 鼓励 | 持续追问会加剧挫败 |
| 紧急纠正（安全相关知识） | 直接纠正 | 不能引导学习者"发现"正确的安全操作 |

## 反馈动作规范

有效的教学反馈应遵循：

| 维度 | 好的反馈 | 差的反馈 |
|---|---|---|
| 具体性 | "你在第三步把分母乘反了" | "再想想" |
| 建设性 | "试试先画一个图来理解" | "这是错的" |
| 及时性 | 在学习者作答后立即 | 拖延到下一天 |
| 鼓励性 | "你的思路方向是对的，只是..." | "你怎么连这个都不会" |
| 过程导向 | "你用了很好的推理方法" | "你真聪明" |

## 默认动作顺序

### 新概念教学

```text
诊断先修知识
  → 激活先验知识（关联已学内容）
  → 讲解核心概念
  → 嵌入式理解检查
  → 练习巩固
  → 反馈与纠正
  → 总结与下一步
```

### 复习与巩固

```text
回忆测试（先测再教）
  → 错误分析
  → 针对性补强
  → 间隔重复调度
```

### 错误概念矫正

```text
诊断性提问（确认错误概念）
  → 制造认知冲突（展示矛盾）
  → 引导重建正确理解
  → 用新问题验证矫正是否成功
  → 记录矫正状态 + 安排后续复查
```
