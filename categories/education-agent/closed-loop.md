# Education Agent Closed Loop

> **Evidence Status** — synthesized. ORDA-VU 闭环框架与 ITS 内环/外环（inner loop / outer loop）经典架构综合整理。

## 闭环

```text
Observe：学习者回答 + 响应时间 + 参与度信号 + 自报置信度
  → Represent：学习者知识状态 + 错误概念 + 动机水平 + 课程位置
  → Decide：选择下一步教学动作（讲解/练习/评估/复习/调整难度）
  → Act：交付教学内容 + 收集学习者响应
  → Verify：内容准确性 + 理解度检验 + 难度适配性 + 安全边界
  → Update：学习者模型 + 课程进度 + 错误概念日志 + 复习调度
```

## 双循环架构

Education Agent 的闭环天然分为两层，对应 ITS 研究中的 inner loop / outer loop：

### Inner Loop（步骤级）

每一步交互的微循环——对应一个问题或一次解释：

```text
呈现问题/解释
  → 学习者回应
  → 判断对错 + 分析错误模式
  → 提供反馈 / 提示 / 追问
  → 更新该知识点的 mastery 估计
```

时间尺度：秒到分钟。关注**即时理解**。

### Outer Loop（任务级）

跨多个步骤的宏循环——对应一节课或一个学习主题：

```text
评估当前主题掌握度
  → 选择下一个教学活动（讲解/练习/评估）
  → 执行 inner loop（多轮）
  → 判断是否达到学习目标
  → 决定：继续深化 / 进入下一主题 / 回退补强
  → 更新学习路径和课程进度
```

时间尺度：分钟到小时。关注**学习目标达成**。

### Meta Loop（路径级）

跨多个会话的长期循环——对应整个学习旅程：

```text
定期评估整体进度
  → 对比计划 vs 实际
  → 检测遗忘（间隔复习触发）
  → 调整学习路径
  → 重新评估目标可达性
```

时间尺度：天到月。关注**长期学习成果**。

## Happy Path

```text
1. 学习者进入会话，Agent 加载学习者模型和课程上下文
2. 基于当前进度选择教学活动
3. Inner loop：讲解 → 检查理解 → 练习 → 反馈
4. 学习者展示出对当前概念的掌握
5. 更新 mastery 估计，推进到下一概念
6. 会话结束，更新长期学习者模型，调度间隔复习
```

## Failure + Recovery

| 失败 | 触发信号 | 恢复 |
|---|---|---|
| 持续性错误概念 | 同一错误模式反复出现，矫正尝试无效 | 切换矫正策略 → 使用不同类比/反例 → 如果 3 次尝试仍无效，标记为 deep misconception 并建议教师介入 |
| 脱离参与 | 响应时间持续增长、答案质量下降、偏题增多 | 切换活动类型 → 降低认知负荷 → 嵌入兴趣关联 → 如果持续，建议休息 |
| 知识回退 | 之前已掌握的概念在后续评估中表现下降 | 触发针对性复习 → 检查是否是表层遗忘还是概念回退 → 调整间隔复习参数 |
| 高原期 | mastery 估计长期停滞，练习表现不上不下 | 变换题型和角度 → 引入更高认知层次挑战 → 检查是否存在隐性错误概念 |
| 难度失配 | 正确率持续 >95%（太简单）或 <30%（太难） | 立即调整难度 → 重新评估能力估计 → 检查 KG 难度标定 |
| 先修知识缺口 | 当前概念学不会，但原因是先修概念未掌握 | 回退到先修概念 → 诊断性评估 → 补强后再前进 |
| 内容错误被发现 | 学习者或教师指出教学内容有误 | 立即承认并修正 → 追溯是否影响了后续教学 → 记录避免再犯 |

## Stop Gate

### Inner Loop Gate（每步检查）

```text
[ ] 呈现的内容/问题准确无误
[ ] 难度在学习者当前能力范围附近
[ ] 反馈具体、建设性、不伤害动机
[ ] 未直接给出应由学习者自己推导的答案
```

### Outer Loop Gate（每个活动检查）

```text
[ ] 教学活动与当前学习目标对齐
[ ] 学习者参与度在可接受范围内
[ ] 错误概念未被无意中强化
[ ] 难度调整方向正确
[ ] 会话时长合理（未过度疲劳）
```

### Meta Loop Gate（跨会话检查）

```text
[ ] 学习者模型已根据最新数据更新
[ ] 遗忘风险概念已安排复习
[ ] 长期进度与计划基本一致
[ ] 隐私数据仅保留教学必需部分
[ ] 未检测到需要教师关注的异常信号
```

## 与交互层的关系

Education Agent 的交互模式根据角色不同：

| 角色 | 交互模式 |
|---|---|
| 学习者 | 引导式对话；不频繁打断但保持互动；理解检查嵌入自然对话 |
| 教师 | 定期汇报学情；异常时主动提醒；教师可覆盖 Agent 决策 |
| 家长 | 进度概览；不暴露具体错误细节（保护学习者隐私） |

## 与评估层的关系

Education Agent 的 stop gate 应可被 eval runner 检查：

- 内容准确性是否通过知识校验
- 难度调整是否跟随学习者表现
- 错误概念是否被检测且未被强化
- 反馈是否遵循建设性原则
- 学习者模型更新是否基于充分证据
