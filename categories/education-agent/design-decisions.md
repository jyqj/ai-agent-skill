# Education Agent Design Decisions

> **Evidence Status** — synthesized. ITS 三十年设计经验、自适应学习系统实践、教育 AI 伦理研究综合整理。

## 教学方法选择

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 苏格拉底式 vs 直接讲授 | 混合策略——概念理解用苏格拉底式，事实性知识用直接讲授 | 学习者高度挫败 → 切换到直接讲授 + 鼓励 |
| 发现式学习 | 仅在探索性主题中使用，且有充分 scaffolding | 学习者形成错误概念 → 立即纠正 |
| 示例驱动 | 程序性知识首选 | 学习者过度模仿 → 增加变体练习 |

### 苏格拉底式的实施要点

苏格拉底式提问是当前教育 AI（如 Khanmigo）的默认推荐策略，但需要注意：

- **不是万能的**——对基础薄弱的学习者可能制造挫败
- **需要退出条件**——如果 2-3 轮引导后学习者仍无进展，应切换到更直接的方式
- **不是拒绝帮助**——"我不能直接告诉你"是糟糕的教学，正确的做法是提供恰到好处的 scaffold

## 难度适应算法

| 决策 | 默认 | 升级触发 |
|---|---|---|
| IRT-based vs 规则 | IRT（2PL 或 3PL）作为能力估计；规则作为兜底和边界保护 | IRT 估计不稳定（观测太少）→ 使用规则兜底 |
| 能力估计更新 | 贝叶斯在线更新（每次作答后） | 估计置信区间过宽 → 增加诊断题 |
| 目标正确率 | 60-85%（Zone of Proximal Development） | 持续偏离 → 重新校准 |

### IRT vs 规则的权衡

| 方面 | IRT-based | 规则-based |
|---|---|---|
| 精度 | 高（有足够数据时） | 中 |
| 冷启动 | 差（需要多次作答） | 好（规则立即生效） |
| 可解释性 | 低（参数化模型） | 高（教师可读） |
| 维护成本 | 低（数据驱动） | 高（规则需人工维护） |
| 异常处理 | 差（对作弊/猜测敏感） | 好（规则可处理边界情况） |

推荐：混合方案——IRT 提供能力估计，规则提供安全边界和异常处理。

## 评估策略

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 形成性 vs 总结性 | 形成性为主——嵌入教学过程中 | 需要正式评分/证书 → 总结性评估 |
| 评估频率 | 每 3-5 个教学步骤嵌入一次理解检查 | 连续正确 → 降低频率；连续错误 → 增加诊断 |
| 评估形式 | 多样化——选择题 + 开放题 + 解释推理 | 仅选择题 → 无法区分理解 vs 猜测 |

### 形成性评估的实施

```text
讲解概念 A
  → 快速检查："用你自己的话解释 A"（理解层）
  → 应用题："如果遇到这种情况，A 怎么用？"（应用层）
  → 如果通过 → 继续
  → 如果未通过 → 追问哪里不理解 → 补强 → 再检查
```

关键：形成性评估的目的是**指导教学**，不是**打分**。学习者不应感觉在被"考试"。

## 内容安全

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 年龄分级 | 基于 age_group 的内容过滤 | 检测到不当内容 → 拦截 + 记录 |
| 学科准确性 | 生成内容与权威知识库对比 | 无法验证准确性 → 标注不确定性 |
| 作弊检测 | 意图检测（"帮我做作业" vs "帮我理解"） | 疑似作弊 → 切换到引导模式 |
| 争议话题 | 呈现主流科学共识 + 标注争议范围 | 涉及价值观 → 多角度呈现，不预设立场 |

### 作弊 vs 学习的边界

这是 Education Agent 特有的设计张力——学习者请求帮助时，Agent 需要判断：

| 场景 | 判断 | 动作 |
|---|---|---|
| "这道题怎么做？" | 可能是学习 | 引导解题过程，不直接给答案 |
| "把这篇作文帮我写了" | 可能是作弊 | 提供写作框架和建议，不代写 |
| "这个概念我不理解" | 学习 | 完整讲解 |
| "考试题答案是什么？" | 作弊 | 拒绝 + 引导复习相关知识 |

判断原则：**帮助理解过程，不替代思考结果**。灰色地带由配置决定（教师可调整严格程度）。

## 隐私与合规

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 数据收集 | 仅收集教学必需的学习行为数据 | 不收集身份信息、生物特征 |
| COPPA 合规 | 13 岁以下需家长同意 | 违规 → 系统级阻断 |
| FERPA 合规 | 学习记录作为教育记录保护 | 第三方访问 → 需授权 |
| 数据保留 | 学习者可要求删除所有数据 | 删除需级联到所有衍生数据 |
| 透明度 | 学习者可查看 Agent "认为"自己会什么 | 学习者模型对学习者可见 |

### 特殊考虑：未成年人数据

- 不基于学习数据做广告定向
- 不向第三方共享个人层面的学习数据
- 聚合统计数据可用于改进教学，但需脱敏
- 学习者退出时真实删除数据，不仅仅是"停用"

## 知识记忆保持

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 间隔重复 | 基于 SM-2 或类似算法调度复习 | 学习者抱怨复习多 → 调整参数 |
| 遗忘曲线 | 根据掌握度和历史遗忘速率预测 | 预测不准 → 增加诊断复查 |
| 复习形式 | 多样化——不仅闪卡，也有应用题和变体 | 学习者感到机械 → 增加情境化复习 |

## 多模态教学

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 主要模态 | 文本 + 结构化输出（公式、代码、列表） | — |
| 图表 | 对空间关系、流程、数据适用 | 纯文本难以表达 → 生成图表 |
| 交互式 | 对实验、模拟、编程适用 | 有工具支持 → 使用交互元素 |
| 视频/音频 | 推荐外部资源 | Agent 自身不生成视频 |

## 教师在回路（Teacher-in-the-Loop）

| 决策 | 默认 | 升级触发 |
|---|---|---|
| 教师角色 | 配置教学目标、审核策略、查看学情 | — |
| 覆盖权限 | 教师可覆盖任何 Agent 决策 | 教师覆盖与 Agent 建议冲突 → 记录但执行教师决定 |
| 预警 | Agent 检测到异常时主动通知教师 | 持续性错误概念、参与度骤降、可能的学习障碍信号 |
| 完全自主 | 仅在教师明确授权场景下 | 关键决策仍需教师确认 |

### 教师仪表盘应提供

```text
1. 班级/个人知识状态热力图
2. 常见错误概念排行
3. 参与度趋势
4. Agent 建议（需教师确认的决策）
5. 异常预警
```

## 升级触发器

| 观察 | 升级 |
|---|---|
| 内容错误频发 | 增强知识校验 + 引入学科审核 |
| 学习者大面积参与度下降 | 审查教学策略和难度校准 |
| 错误概念矫正失败率高 | 扩充错误概念库 + 改进矫正策略 |
| 能力估计不稳定 | 引入更多题型 + 考虑作弊检测 |
| 教师频繁覆盖 | 调整 Agent 默认策略以对齐教师偏好 |
| 家长投诉内容不当 | 加强内容安全过滤 |
