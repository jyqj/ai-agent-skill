# Enterprise Workflow Agent Task Model

> **Evidence Status** — synthesized. enterprise BPM 系统、审批引擎、ServiceNow/Salesforce workflow 模式。

## 任务分类矩阵

| 任务类型 | 默认深度 | 默认自治 | 成功定义 | 典型失败模式 |
|---|---|---|---|---|
| 工单路由/分派 | D4 | L4 | 正确分配 + 通知送达 | 错误分派、通知遗漏、规则过期 |
| 审批链推进 | D4-D5 | L3-L4 | 审批完成 + 状态更新 | 超时未升级、审批人错误、并发冲突 |
| 跨系统数据同步 | D4 | L4-L5 | 源目一致 + 冲突解决 | schema 不匹配、部分写入、幂等性破坏 |
| 合同/发票处理 | D5 | L3 | 提取→校验→录入→审批→付款 | OCR 错误、金额不匹配、重复处理 |
| 员工入/离职流程 | D5-D6 | L3 | 多系统联动完成 | 遗漏系统、权限残留、时序依赖 |
| SLA 监控与升级 | D4 | L4-L5 | 违约检测 + 通知 + 补偿 | 误报、升级链断裂、时区错误 |
| 报表/合规报告 | D4 | L4 | 数据准确 + 格式合规 + 按时提交 | 数据延迟、格式错误、遗漏字段 |
| 客户投诉处理 | D5 | L3 | 记录→分派→跟进→解决→回访 | 分派错误、跟进超时、解决不彻底 |

## 任务深度说明

- D4（执行并验证）：大多数标准流程步骤。执行写动作后必须 read-back 验证。
- D5（多步编排）：跨多个系统的联动流程。需要 checkpoint、补偿事务、中间状态管理。
- D6（长期追踪）：跨天/跨周的流程（如合同审批、入职流程），需要 durable execution 和 SLA 时钟。

## 任务组合模式

企业流程通常由多步任务链组成：

```text
触发 → 验证前置条件 → 执行核心动作 → 验证效果
  → [成功] 推进到下一步
  → [失败] 补偿 → 升级 → 等待人工介入
  → [超时] 提醒 → 升级 → 替代路径
```

## 复杂度维度

| 维度 | 低复杂度 | 高复杂度 |
|---|---|---|
| 系统数量 | 单系统 CRUD | 3+ 系统联动 |
| 审批层级 | 无审批或单级 | 多级审批 + 条件分支 |
| 数据敏感度 | 内部运营数据 | 财务/PII/合规数据 |
| 时间跨度 | 分钟级完成 | 跨天/跨周 |
| 回滚复杂度 | 单步可逆 | 需要多步补偿事务 |
| 并发风险 | 独占实体 | 多人/多流程竞争同一实体 |
