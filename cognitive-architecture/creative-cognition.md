# 创意认知——Agent 如何产生新方案

> **Evidence Status** — theoretical. 来自创造力研究（Boden 三类创造力、TTCT）、LLM 创造力评估（arXiv 2604.13242）和多 Agent 创意协作实践。

## 1. 为什么需要这篇

`beyond-verification.md` 讨论了如何评估创意输出，但没有讨论"Agent 如何产生新方案"的认知机制。理解创意认知有助于：

- 设计 **Creative Agent** 的核心认知管线
- 改进**所有品类 Agent** 的方案生成能力（Coding Agent 生成多种实现方案、Research Agent 提出新假设）
- 明确当前 LLM 的创造力边界在哪里、可以如何工程性地扩展

## 2. 创造力的认知分类（Boden）

Margaret Boden 将创造力分为三类，LLM Agent 在这三类上的表现差异显著：

### 2.1 组合创造力（Combinatorial）

将已有概念以新方式组合——这是 LLM 的核心能力。

```text
示例：将 REST API 设计模式 + 事件溯源 + CQRS → 新的微服务架构方案
能力：LLM 在训练语料中见过大量概念，组合能力强
局限：组合的"新颖性"受限于训练数据中概念的共现频率
```

### 2.2 探索创造力（Exploratory）

在概念空间内系统搜索——Plan-and-Execute + 多方案对比。

```text
示例：在"所有可能的排序算法"空间中搜索适合特定约束的变体
能力：通过 prompt 引导和多步推理，LLM 可以系统性地探索设计空间
局限：搜索深度受限于上下文窗口和推理步数
```

### 2.3 变革创造力（Transformational）

改变概念空间本身——当前 LLM 的根本局限。

```text
示例：发明新的编程范式（如从过程式到函数式的跳跃）
能力：当前 LLM 几乎无法做到
原因：变革性创造要求质疑并修改概念空间的基本假设，
      而 LLM 的推理在训练分布定义的概念空间内运作
```

**当前 LLM 展现组合和探索性创造力，无法达到变革性创造力**（arXiv: On the Creativity of AI Agents）。工程设计应围绕前两类做优化，而非期望第三类。

## 3. 发散与收敛

创意过程不是单一方向的生成，而是发散和收敛的交替：

### 3.1 发散思维（Divergent Thinking）

生成多个候选方案——追求数量和多样性。

```text
工程手段：
  - temperature 调高：增加输出随机性
  - 多 Agent 竞争：不同 Agent 独立生成方案
  - prompt 变体：同一任务用不同 prompt 框架（角色、约束、角度）
  - 约束放松：暂时移除某些假设（见 §5）
```

### 3.2 收敛思维（Convergent Thinking）

从多个候选中选择最佳——追求质量和一致性。

```text
工程手段：
  - evaluation rubric：多维评分标准
  - 投票：多 Agent 或多模型投票
  - A/B 测试：对用户呈现不同方案并收集偏好

注意：Agent 创意 A/B 测试需要 10,000+ 轨迹/臂才能达到统计意义
     （因非确定性输出导致方差远大于传统 A/B）
```

### 3.3 双菱形模型

创意任务通常经历两轮发散-收敛：

```text
第一菱形（问题空间）：
  发散：探索问题的多种理解方式
  收敛：确定要解决的核心问题

第二菱形（方案空间）：
  发散：生成多种解决方案
  收敛：选择并精炼最佳方案
```

## 4. 创意 Agent 的架构模式

### 4.1 多角色管线

```text
Research → Outliner → Writer → Editor → Fact-Checker

每个角色专注一个认知阶段：
  Research  — 信息收集（发散）
  Outliner  — 结构设计（收敛）
  Writer    — 内容生成（发散）
  Editor    — 质量优化（收敛）
  Fact-Checker — 准确性验证（收敛）
```

### 4.2 多模型联合体

专业化 LLM 独立生成 + 推理 Agent 综合：

```text
Model A（擅长叙事）→ 方案 1
Model B（擅长分析）→ 方案 2     → Synthesizer Agent → 最终方案
Model C（擅长创意）→ 方案 3
```

### 4.3 竞争-合作平衡

- **竞争**刺激发散：多个 Agent 独立生成方案，互不可见
- **合作**促进精炼：Agent 之间交换方案并相互改进

关键平衡点：纯竞争导致冗余；纯合作导致趋同。最优策略通常是"先独立生成，再协作精炼"。

### 4.4 风格一致性

跨输出和跨会话保持一致风格是创意 Agent 的核心挑战：

```text
四层防线：
  1. System Prompt — 定义风格基线（粗粒度）
  2. Fine-Tuning — 训练风格偏好（中粒度）
  3. Memory — 存储风格规则和历史示例（细粒度）
  4. Post-Processing — 后处理风格校正（兜底）

StyleAligned 方法：通过注意力共享保持跨输出一致性
```

## 5. 约束放松与类比迁移

### 5.1 约束放松

创造力常来自临时移除某些"理所当然"的假设：

```text
示例：
  约束："API 必须是同步的" → 放松后 → 探索异步/事件驱动方案
  约束："数据必须存在数据库" → 放松后 → 探索内存/流式处理方案

实现：在 prompt 中显式列出当前假设，然后要求 Agent 逐个移除并探索后果
```

### 5.2 类比迁移

从不同领域映射解决方案——参见 `reasoning-modes.md` 中的类比推理。

```text
示例：
  源域：生物免疫系统的分布式检测
  目标域：网络安全的入侵检测
  迁移：自适应、分布式、无中心的检测机制

实现：在 prompt 中提供源域描述，要求 Agent 找到结构性映射
```

### 5.3 与元认知的关系

元认知控制决定"何时应该打破常规思路"（参见 `metacognitive-control.md`）：

```text
信号：常规方法连续失败 → 元认知触发 → 约束放松或类比迁移
信号：方案空间过于同质 → 元认知触发 → 增加发散（温度、角色变体）
```

## 6. 创意评估（补充 beyond-verification.md）

### 6.1 功能性创造力 vs 本体论创造力

（arXiv 2604.13242）

```text
功能性创造力：可观察输出是否具有创造性？
  → 可以通过人类评估者判断
  → 关注产品（product）

本体论创造力：底层过程是否具有创造性？
  → 需要理解生成机制
  → 关注过程（process）

当前评估大多关注功能性创造力，因为本体论创造力难以观测。
```

### 6.2 评估方法

| 方法 | 适用场景 | 局限 |
|---|---|---|
| Amabile 共识评估 | 多专家独立评分后取共识 | 成本高、耗时长 |
| 代码评分器 | 可量化指标（新颖性分数、多样性分数） | 无法捕捉主观维度 |
| LLM 评分器 | 主观标准（风格、情感影响） | 存在偏见和不一致 |
| 混合评估 | 生产环境 | 需要校准代码和 LLM 评分的权重 |

### 6.3 关键维度

- **创造力**：输出是否超越常规模式？
- **新颖性**：在目标受众的参考框架中是否为新？
- **连贯性**：内部逻辑和结构是否一致？
- **相关性**：是否回应了原始意图或约束？
- **情感影响**：是否引发预期的情感响应？（适用于文学、艺术类）

## 7. 与知识库的映射

| 创意认知概念 | 对应 Plane/文件 | 设计启发 |
|---|---|---|
| 发散生成 | Orchestration（多 Agent 竞争） | temperature 和 prompt 变体增加多样性 |
| 收敛选择 | Control（judge/vote） | 评估 rubric + 投票机制 |
| 风格一致性 | Representation, Memory | 风格锚定 + 记忆中的风格规则 |
| 约束放松 | Metacognitive Control | 元认知触发"打破常规"策略 |
| 类比迁移 | Reasoning Modes | 类比推理的显式触发 |

## 8. 设计检查清单

- [ ] 创意管线是否包含明确的发散和收敛阶段？
- [ ] 是否使用多 Agent 或多 prompt 变体增加候选多样性？
- [ ] 风格一致性如何保证（跨输出、跨会话）？
- [ ] 评估标准是否区分了功能性创造力和主观偏好？
- [ ] 是否有安全边界防止创意输出违反价值约束？
