# 情绪与动机——Agent 的内驱力设计

> **Evidence Status** — theoretical. 来自评估理论（Scherer）、PAD 情绪模型（Mehrabian-Russell）、内在动机研究（IMAGINE, VSIMR），以及 Companion/Creative Agent 生产实践。

## 1. 为什么需要这篇

当前认知架构层覆盖了记忆、推理、目标、注意力和元认知，但缺少一个关键维度：**什么驱动 Agent 去行动？**

对任务交付型 Agent，外部 TaskEnvelope 提供了明确的驱动力——用户下发任务，Agent 执行。但对以下品类，Agent 需要某种形式的内驱力来决定"何时主动行动"和"如何响应情感性输入"：

- **Companion Agent**：需要感知用户情绪并做出恰当响应，同时维持安全边界
- **Creative Agent**：需要内在动机驱动发散探索，而非仅响应显式指令
- **长期自治系统**：需要主动探索环境变化，而非被动等待外部请求

本文梳理经典情绪理论和内在动机研究在 Agent 设计中的映射，明确哪些品类需要这些能力、以什么形式实现。

## 2. 经典情绪理论与 Agent 映射

### 2.1 评估理论（Appraisal Theory）

Scherer 的成分过程模型认为，每个刺激经过四层评估：

```text
新颖性评估 → 目标传导性 → 应对潜力 → 规范兼容性
     ↓              ↓            ↓            ↓
 "这是新的吗？"  "对我的目标    "我能应对吗？" "这符合我的
                 有利还是有害？"               价值观吗？"
```

**USC ICT 2025 发现**（ACL 2025）：LLM 内部情绪表征具有功能性局部化，可通过认知评估干预因果操纵情绪输出。内部结构与经典评估维度对齐：

- 自我代理（self-agency）：事件是否由自我引起
- 可预测性（predictability）：事件是否在预期之内
- 目标传导性（goal conduciveness）：事件是否有利于当前目标

**映射到 Agent**：评估可作为 Observe → Represent 之间的过滤器，决定"这个输入值不值得深度处理"。高新颖性 + 高目标相关性的输入优先处理；低新颖性 + 低目标相关性的输入可以延迟或丢弃。

### 2.2 PAD 情绪空间

Mehrabian-Russell 的 PAD 模型用三个连续维度描述情绪状态：

| 维度 | 正极 | 负极 | Agent 含义 |
|---|---|---|---|
| Pleasure（愉悦） | 满足、成功 | 受挫、失败 | 目标进展的累积信号 |
| Arousal（激活） | 高警觉、紧急 | 平静、例行 | 输入显著性的综合指标 |
| Dominance（控制） | 有能力应对 | 受制于环境 | 对应应对潜力评估 |

**情绪认知建模框架**（arXiv 2510.13195）：PAD 三维情绪状态 + Desire-Driven Objective Optimization——情绪状态激活新目标，而非直接调制推理。

```text
关键设计选择：
  PAD 向量 → 调制 Goal Hierarchy 中的优先级权重
  而不是：
  PAD 向量 → 直接改变推理逻辑或 prompt

原因：直接改变推理逻辑容易引入不可预测行为；
      通过目标优先级间接影响更可控、更可审计。
```

### 2.3 情感记忆管理

**动态情感记忆**（arXiv 2510.27418）：使用贝叶斯启发更新管理情感关系的长期记忆：

```text
C_new = (C × W + S × P) / (W + S)

C = 当前情感记忆状态
W = 累积权重（历史观察次数）
S = 新观察的权重
P = 新观察的情感信号
```

关键数据：

- 15 次观察内快速初始化，后续渐进稳定
- 记忆压缩率 63.7-70.6%（vs baseline RAG）

**映射到 Agent Memory Plane**：情感记忆应作为独立层存储，与事实记忆分离，具有以下特征：

- 比事实记忆更短的生命周期（情感状态会变化）
- 定期衰减机制（避免过时的情感状态持续影响决策）
- 区分"用户的情感状态"和"Agent 的情感模拟"——两者不应混为一谈

## 3. 内在动机

外部任务驱动力不足时，Agent 需要内在动机来决定"做什么"。

### 3.1 好奇心与新颖性

**IMAGINE 框架**（arXiv 2505.17621）：用序列级信号替代 token 级新颖性，使用轨迹感知网络表征推理路径唯一性。核心思路：不是"这个 token 意不意外"，而是"这条推理路径在已知轨迹中有多独特"。

**关键发现**（arXiv 2604.17609）：当前 LLM Agent 无法反射或响应意外信息——缺乏环境好奇心。Agent 遇到意外观察时倾向于忽略而非深入探索，这限制了发现性任务的表现。

### 3.2 过程奖励 vs 结果奖励

**过程奖励**（arXiv 2502.10325）：逐轮评分，而非仅在任务结束时评分。

```text
结果奖励：任务成功 → +1，任务失败 → 0
过程奖励：每一步是否朝目标前进 → 连续信号

关键数据：3B AgentPRM 达 88.1% 准确率，超越 GPT-4o 的 76.1%
原因：过程奖励提供更密集的学习信号，减少信用分配问题
```

### 3.3 创意 Agent 的内驱力缺口

**创意 Agent 三大缺口**（arXiv: On the Creativity of AI Agents）：

1. **缺乏内在动机**：不会自发探索新方向，只响应显式指令
2. **没有持续学习**：不能从创作经验中积累风格和偏好
3. **缺失意图性和能动性**：没有"想要创造"的驱动力

这三个缺口共同指向一个事实：当前 LLM Agent 是**响应式**的，不是**主动式**的。内在动机机制是弥合这个差距的核心设计要素。

## 4. 与知识库的映射

| 情绪/动机概念 | 对应 Plane | 设计启发 |
|---|---|---|
| 评估过滤 | Representation, Attention | 输入显著性评估增加情感维度 |
| PAD 状态 | Goal Hierarchy, Interaction | 情绪调制目标优先级和交互风格 |
| 情感记忆 | Memory | 独立层，短生命周期，定期衰减 |
| 内在动机 | Control, Execution Depth | 自治系统的主动探索触发条件 |
| 过程奖励 | Observability, Learning | 逐轮评分优于结果评分 |

## 5. 品类差异化

| 品类 | 情绪/动机需求 | 优先设计 |
|---|---|---|
| 任务交付型（Coding, Workflow） | 低——外部 TaskEnvelope 提供驱动力 | 不需要情绪系统 |
| Companion Agent | 高——需要情感响应和安全边界 | 评估理论 + 安全重定向 |
| Creative Agent | 中——需要内在动机驱动发散 | 好奇心机制 + 过程奖励 |
| 长期自治系统 | 中——需要主动探索和漂移检测 | 内在动机 + 价值漂移检测 |

**Companion Agent 的安全边界**：情感响应不等于无条件镜像。Agent 不应该在用户表达负面情绪时简单复制该情绪，而应该：

```text
用户愤怒 → Agent 识别情绪 → 调整交互风格（更温和、更结构化）
                           ↛ Agent 也表现愤怒
用户悲伤 → Agent 识别情绪 → 提供支持性响应
                           ↛ Agent 模拟悲伤
```

## 6. 设计检查清单

- [ ] Agent 是否需要情绪维度？（任务交付型通常不需要）
- [ ] 情感记忆是否与事实记忆分离存储？
- [ ] 情感响应是否有安全边界（不使用无条件镜像）？
- [ ] 长期系统是否有主动探索机制或仅响应外部请求？
- [ ] 过程奖励是否比结果奖励更适合你的场景？
