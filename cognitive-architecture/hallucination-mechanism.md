# 幻觉机制——工作记忆中证据不足时的填充行为

> **Evidence Status** — synthesized | 从认知架构角度解释 LLM 幻觉的产生机制和防御策略。

## 认知机制

LLM 的幻觉可以从工作记忆模型理解：当 Context Window 中缺乏充分证据支撑某个推断时，模型倾向于用**概率性填充**（probabilistic confabulation）补全缺口——生成看似合理但无事实依据的内容。这不是随机错误，而是默认的"最大似然补全"行为。

关键触发条件：
- **检索缺失**：应该检索外部证据但未执行检索，模型用内部参数知识填充
- **证据模糊**：检索到的信息不足以确定性回答，模型过度推断
- **指令压力**：Prompt 要求"给出答案"而非"说明不确定"，迫使模型补全

## 防幻觉策略的认知层面指导

### 1. 增加检索证据

在推理链中设置**强制检索点**：当推断依赖的事实不在当前 Context 中时，必须先检索再回答，而非依赖参数记忆。这对应 Working Memory 的"外部刷新"——用环境信息替代内部猜测。

### 2. 显式不确定性标注

允许 Agent 输出"不确定"或"证据不足"。认知层面：降低补全压力，让模型在证据不足时选择"不回答"而非"编造回答"。工程实现：在 System Prompt 中显式授权不确定性表达。

### 3. Verification Gate

在 Effects Plane 的操作执行前设置验证门：Agent 生成的事实性断言必须经过可验证性检查（能否通过工具调用确认？），不可验证的断言应标注置信度。
