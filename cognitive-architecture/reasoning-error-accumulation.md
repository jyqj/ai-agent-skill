# 推理误差累积——CoT 链越长，风险越大

> **Evidence Status** — synthesized | 从认知架构角度讨论多步推理中的误差传播及其对 Agent 设计的影响。

## 核心问题

多步推理（CoT、ReAct、Plan-Execute）中，每一步都有非零的错误概率。步骤串联时，错误概率不是线性叠加而是乘法累积：

```text
单步正确率 95% -> 10 步后整体正确率 = 0.95^10 = 60%
单步正确率 90% -> 10 步后整体正确率 = 0.90^10 = 35%
```

这意味着长推理链的可靠性会急剧下降，即使每一步看起来都"差不多对"。

## 对 Agent 设计的实际影响

### 1. 步骤数上限

推理链不应无限延长。设计时应设定步骤预算，超过预算时强制进入总结-验证-重规划循环，而非继续延伸链条。

### 2. 中间验证点

在长推理链中插入**验证检查点**：每 N 步暂停，用工具调用或自验证确认中间结果的正确性。错误越早发现，修正成本越低。

### 3. 分治策略

将长链拆分为独立的子任务，每个子任务有明确的输入/输出契约和独立验证。子任务之间通过验证过的中间结果传递，而非依赖连续的推理链上下文。

### 4. 误差源头管理

主要误差来源及对策：
- **幻觉填充**（见 `hallucination-mechanism.md`）：强制检索替代参数记忆
- **上下文丢失**：长链中早期信息被挤出窗口，用 pin 或摘要保持关键前提
- **错误传播**：前序步骤的错误被后续步骤当作事实，中间验证点可截断传播链
