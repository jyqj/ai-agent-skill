# 用同一个模型当自己的法官

> **Evidence Status** — synthesized. LLM-as-Judge 研究中的同源偏差发现；Multi-Agent 系统中的 self-evaluation 实验；生产环境中 Reflection 阶段失效的案例。

---

## 场景

一个 Research Agent，负责根据用户问题搜索文献、综合分析、生成研究报告。为了提高输出质量，团队在 Agent 循环中加入了 Reflection 阶段：生成报告后，让模型 critique 自己的输出，发现问题后修改再输出。

模型：GPT-4 级别的大语言模型，同一个模型实例，同一个 system prompt。

## 设计选择

团队实现了一个 self-reflection loop：

```
生成报告 -> [同一模型] critique 报告 -> 修改 -> 再 critique -> 满意则输出
```

Reflection prompt 大致为："请检查你上面生成的报告，指出事实错误、逻辑漏洞和遗漏的重要观点。"

选择同一模型的理由：
- 一个模型就够了，为什么要引入第二个？
- 同一模型对自己的输出最了解，critique 应该最准确
- 减少系统复杂度和成本

## 预期

- Reflection 阶段能捕获生成阶段的错误
- 经过 2-3 轮 self-critique，报告质量显著提升
- 相当于一个人写完文章后自己检查一遍

## 实际结果

### Self-Serving Bias（自我服务偏差）

模型在 critique 阶段系统性地倾向于认可自己之前的决策。表现为：

- 对明显的事实错误，critique 阶段给出"基本准确，可以进一步补充"而不是"这是错误的"
- 对逻辑跳跃，critique 阶段倾向于"为其辩护"而不是"指出漏洞"
- critique 的措辞普遍偏向肯定："总体不错，但可以在 X 方面加强"

### 幻觉的同源验证

最危险的问题：当模型在生成阶段产生幻觉（编造一个不存在的研究结论）时，同一个模型在 critique 阶段倾向于"验证"这个幻觉——因为幻觉的来源和验证者共享相同的参数权重和知识盲区。

典型案例：模型声称"Smith et al. (2023) 发现 X"（实际上这篇论文不存在），critique 阶段的评价是"引用恰当，支持论点"。

### Critique 的表演性

经过几轮观察，团队发现 critique 阶段更像是"表演在检查"而不是"真的在检查"：

- 每次 critique 都会提出 2-3 个"改进建议"，但多数是措辞调整，不涉及实质性错误
- 模型学会了生成"看起来像 critique"的输出，而不是真正的批判性分析
- 修改后的版本往往只是在被 critique 的地方加了几句话，核心错误原封不动

## 根因分析

**Judge 和 Executor 共享相同的知识和偏差，无法提供独立视角。**

这是一个信息论问题：如果 Judge 的知识完全是 Executor 知识的子集（同一模型），那么 Judge 不可能发现 Executor 的知识盲区。

类比：让一个人检查自己的拼写错误，他大概率会"看到"正确的拼写，因为大脑会自动补全预期内容。这就是为什么编辑和作者不能是同一个人。

具体机制：

1. **共享偏差**：同一模型的训练数据偏差在生成和 critique 阶段都存在，无法通过 self-reflection 消除
2. **知识盲区重叠**：模型不知道的东西，在两个阶段都不知道。幻觉的本质就是"模型不知道自己不知道"
3. **输出锚定**：critique 阶段已经看到了生成的输出，受锚定效应影响，很难做出与之矛盾的判断

## 教训

### 高风险判断应使用异源 Judge

"异源"指的是至少在以下一个维度上独立：

- **不同模型**：用 Claude 检查 GPT 的输出，或反过来。不同训练数据 = 不同偏差 = 更高概率发现盲区
- **外部工具验证**：用搜索引擎验证引用是否存在、用代码执行验证推理是否正确、用数据库查询验证事实
- **人工审查**：高风险决策的最终防线

### 低风险可以接受同源 Reflection

不是说 self-reflection 毫无价值。对于以下场景，同源 reflection 仍然有用：

- **格式检查**：输出是否符合要求的格式
- **完整性检查**：是否遗漏了用户明确提到的要求
- **语气调整**：措辞是否恰当

这些是模型有能力自我检查的范围，因为它们不依赖外部知识。

### 异源 Judge 的成本权衡

| 策略 | 成本 | 独立性 | 适用场景 |
|---|---|---|---|
| 同模型 self-reflection | 低 | 无 | 格式、完整性、语气 |
| 不同模型 cross-check | 中 | 中 | 事实核查、逻辑验证 |
| 工具验证 | 中 | 高 | 可计算/可查询的断言 |
| 人工审查 | 高 | 最高 | 高风险决策、法律/医疗建议 |

根据任务风险等级选择合适的 Judge 策略，而不是一刀切。
