# Feedback Reports

> **Evidence Status** — theoretical. 这是反馈报告归档目录的占位说明；真实反馈报告应在产生后单独声明 Evidence Status。

## 用途

本目录存放使用者或维护者提交的结构化反馈，用于支撑知识库内容的 Evidence Status 升级、降级或修正。

文件命名建议：

```text
YYYY-MM-DD-<project-slug>.md
```

每份报告至少应说明：

- 使用了哪个 skill 文件；
- 使用场景和 Agent 品类；
- 结果是 useful / missing / misleading / outdated 中哪类；
- 是否有可回查证据、评估结果或生产反馈。

模板见 `../feedback-template.md`；升降级标准见 `../evidence-upgrade-criteria.md`。
