# NagaAgent 四层 Prompt 系统


> **Evidence Status** — grounded. 本目录下的项目分析、源码片段或专题笔记。

## 分层设计理念

AI 角色系统需要管理多维度信息，但这些维度的**生命周期和变更频率完全不同**：

| 维度 | 变更频率 | 所有权 | 位置 |
|-----|--------|--------|------|
| 角色人格 | 极低 | 角色模板 | Tier1 |
| 工具规则 | 低 | 系统平台 | Tier2 |
| 干员灵魂 | 中 | 干员实例 | Tier3 |
| 运行时上下文 | 高 | 会话级 | Tier4 |

**四层分层方案**：按变更周期和所有权分离存储、装配、注入。

```
┌─────────────────────────────────────────────────────────────┐
│ LLM 最终 System Prompt（由四层装配而成）                    │
├─────────────────────────────────────────────────────────────┤
│ Tier4: 动态上下文（时间、技能、MCP、RAG、搜索、激活技能）  │
├─────────────────────────────────────────────────────────────┤
│ Tier3: 实例灵魂（SOUL.md、记事本、长期记忆）               │
├─────────────────────────────────────────────────────────────┤
│ Tier2: 工具调度规则（MCP边界、多干员协作、执行器边界）    │
├─────────────────────────────────────────────────────────────┤
│ Tier1: 基础人格（平台根提示+角色人设+内置技能）            │
└─────────────────────────────────────────────────────────────┘
```

---

## Tier1：基础系统提示层

**定位**：不轻易变化的基础人格和角色身份

包含内容（按装配顺序）：

1. **平台根提示词** (**1_平台根提示词**)
   - 来源：`system/prompts/platform_root_prompt.txt`
   - 职责：定义平台所有角色的通用基础范式

2. **角色人格与对话风格** (**2_角色人格引用槽**)
   - 注入变量：`{character_identity_prompt}`, `{character_conversation_style_prompt}`
   - 来源：`characters/{角色名}/conversation_style_prompt.txt`

3. **角色内置技能** (**3_角色内置能力引用槽**)
   - 注入变量：`{character_builtin_skills_prompt}`
   - 来源：`characters/{角色名}/skills/*/SKILL.md`

**关键设计**：
- Tier1 对应 `system_prompt` 字段
- 加载后**禁止运行时修改**（除非切换角色）
- 干员后天发展**必须**放在 Tier3

---

## Tier2：基础工具层

**定位**：工具、调度、协作边界的平台级规则

包含内容：
1. **工具调度总则** - 工具调用格式、JSON schema、错误处理
2. **直接工具与 MCP 边界** - 区分直接工具 vs MCP
3. **多干员协作规则** - `agents_list`、`agent_relay` 使用规范
4. **OpenClaw 临时执行器边界** - 防止直接调度 OpenClaw

**关键特点**：
- Tier2 进入每次会话的独立 system 消息
- 如果**切换引擎**，工具规则**必须重建**
- Tier2 内容对**所有干员通用**

---

## Tier3：实例灵魂与记忆层

**定位**：干员的后天发展、私有成长、实例级状态

包含内容：

1. **实例灵魂** (**1_实例灵魂引用槽**)
   - 来源：`.naga/agents/{干员ID}/SOUL.md`
   - 职责：干员的个性化发展、长期倾向、独特记忆

2. **实例记事本** (**2_实例记事本引用槽**)
   - 来源：`.naga/agents/{干员ID}/notes/CLAUDE.md`
   - 职责：短期工作注意事项、项目进度、待做事项

3. **实例长期记忆** (**3_实例长期记忆引用槽**)
   - 来源：`.naga/agents/{干员ID}/memory/*.md` 或 NagaMemory
   - 职责：长期知识库和历史经验摘录

**关键设计**：
- 每次从**实例文件**动态读取
- 只对当前干员有效，不影响其他干员
- 灵魂更新**不回写** Tier1

---

## Tier4：动态运行时上下文层

**定位**：会话级别的动态信息，每次对话都可能变化

包含内容：

| 文件 | 注入变量 | 职责 |
|-----|---------|------|
| **1_时间与环境** | `{time_info}` | 当前日期、时间、环境 |
| **2_技能目录** | `{skills_section}` | 当前可用技能列表 |
| **3_MCP目录** | `{available_mcp_tools}` | 当前可用 MCP 服务 |
| **4_当前身份与干员通讯录** | `{multi_agent_context_section}` | 干员身份、协作指引 |
| **5_前置搜索结果** | `{search_section}` | 搜索结果摘录 |
| **6_RAG记忆注入** | `{rag_section}` | 相关历史记忆 |
| **7_激活技能指令** | `{skill_active_section}` | 激活技能的完整指令（最高优先级） |

**关键设计**：
- 所有内容**运行时动态生成**
- 每次会话都**可能不同**
- 是**不稳定内容**的集中地

## 渐进披露策略

**按需激活原则**：

| 信息类型 | 何时注入 | 何时不注入 |
|---------|--------|---------|
| 技能目录 | `include_skills=True` | 用户明确选择了技能 |
| 工具指令 | `include_tool_instructions=True` | LLM 支持原生 function calling |
| 搜索结果 | 有搜索结果时 | 搜索未执行或为空 |
| RAG 记忆 | 记忆库有召回时 | 无相关结果 |
| 激活技能 | 用户明确指定 | 技能未选择 |

**优先级规则**（从高到低）：
1. 用户明确指定的技能（激活技能指令）
2. Tier1 人格（基础身份）
3. 工具调度规则（平台约束）
4. 干员灵魂和记忆（上下文）
5. 时间、技能目录、MCP 列表（环境信息）

## 三层存储原则

```
system/prompts/        → 平台级固定模板
characters/{角色名}/    → 角色模板层
.naga/agents/{干员ID}/ → 干员成长层
```

**原则**：
1. `system/prompts` 不放角色私有或干员私有内容
2. `characters` 不放干员后天发展或实例私有内容
3. `.naga/agents/{id}` 只存放该干员的私有演进

## 模板变量渲染

**占位符模式**：`{variable_name}`

```python
def _render_prompt_template(template: str, variables: Dict) -> str:
    # 正则替换：{variable_name} -> variables["variable_name"]
    rendered = _PROMPT_SLOT_PATTERN.sub(
        lambda match: str(variables.get(match.group(1), "") or ""),
        template,
    )
    # 清理多余空行
    rendered = re.sub(r"\n{3,}", "\n\n", rendered)
    return rendered.strip()
```

**变量命名约定**：
| 前缀 | 含义 | 示例 |
|-----|-----|------|
| `platform_` | 平台级 | `platform_root_prompt` |
| `character_` | 角色级 | `character_identity_prompt` |
| `agent_` | 干员实例级 | `agent_soul_prompt` |
| `available_` | 可用资源列表 | `available_mcp_tools` |

## 引擎切换策略

**保留**（跨引擎）：
- `system/prompts` 中的平台固定层
- `characters` 中的角色模板
- 同干员的 `.naga` 灵魂与成长数据

**替换**（引擎特定）：
- Tier2 工具调度规则
- 引擎私有 bootstrap 结构
- 原始工具轨迹和执行结果

策略：
- **同角色、同干员、换引擎** → 保留灵魂，重建工具层
- **同角色、不同干员** → 保留角色模板，不共享灵魂
- **不同角色** → 完全重建 Tier1
